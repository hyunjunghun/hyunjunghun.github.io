## 쌀먹

처음부터 AI 모델을 직접 만들고 실행하는 것은 현실적으로 어렵다. 모델 학습에는 많은 데이터와 GPU 자원이 필요하고, 강의자료처럼 텍스트·수식·그림이 섞인 자료를 처리하려면 더 복잡한 모델이 필요하다.

따라서 초기에는 직접 모델을 만드는 방식보다는, 이미 제공되는 AI API를 활용해서 기능을 구현하려고 한다. 우선 API를 사용해 강의자료 페이지를 입력하면 AI가 내용을 설명해주는 기본 흐름을 만들고, 이후 필요에 따라 오픈소스 모델이나 자체 모델 활용 가능성을 검토할 예정이다.

## 사용할 API 선택

사용할 API는 OpenAI API로 정했다. 평소 공부할 때 ChatGPT를 자주 사용하고 있어서, 비슷한 방식으로 익숙하게 공부할 수 있을 것 같았기 때문이다.

![OpenAI API 가격표](/assets/img/posts/drop_class/openai_api_price_batch.png)

OpenAI API에는 Standard, Batch, Data Residency 옵션이 있다. 이 프로젝트에서 다루는 강의자료는 중요한 기밀 정보나 민감한 데이터가 아니기 때문에 Data Residency는 우선 고려하지 않았다.

따라서 실제로 고려할 대상은 Standard와 Batch이다.

처음에는 Standard 방식과 Batch 방식 중 어떤 것을 사용할지 고민했다. Standard 방식은 요청을 보내면 바로 응답을 받을 수 있다는 장점이 있지만, 가격이 상대적으로 비싸다. 반면 Batch 방식은 미리 강의자료를 업로드하고 처리 결과를 나중에 받는 방식이기 때문에 즉시성은 떨어지지만, 가격이 더 저렴하다.

이 프로젝트에서는 사용자가 공부하는 중에 계속 질문하고 답변을 받는 구조까지는 구현하지 않을 생각이다. 그렇게 하면 매 질문마다 토큰이 계속 소모되고, 이전 대화 맥락까지 포함되면서 비용이 감당하기 어려워질 수 있기 때문이다.

대신 목표는 PPT나 PDF 강의자료를 미리 업로드하면, 각 페이지에 대한 설명을 생성해두는 것이다. 이후 추가 질문이 필요하면 개인적으로 ChatGPT 같은 서비스를 이용하면 된다.

따라서 현재 구조에서는 실시간 응답이 중요한 Standard 방식보다, 비용을 줄일 수 있는 Batch 방식이 더 적합하다고 판단했다.
